## Что уже работает

### Источники

- YouTube и YouTube Shorts.
- Публичные Instagram Reels через авторизованные cookies.
- Единый конвейер обработки YouTube и Reels.

### Обработка

- Скачивание MP4 и MP3.
- Автоматическое преобразование Reels в совместимый H.264/AAC MP4.
- Автоматическое сжатие файлов больше лимита Telegram до размера менее 49 МБ.
- Расшифровка Faster Whisper с таймкодами.
- Markdown-транскрипт.
- Summary через YandexGPT.
- Полный построчный перевод через Yandex Translate.
- Тренерский перевод с сохранением английских терминов и словарём.
- Русские субтитры `.srt`.
- MP4 со встроенными русскими субтитрами.
- Русская озвучка через Yandex SpeechKit.
- Живое сообщение прогресса для долгих операций.

### Надёжность

- При новой ссылке очищается кеш предыдущего видео.
- Количество строк и таймкодов полного перевода сохраняется.
- Ответы-отказы YandexGPT не сохраняются как готовый результат.
- Тяжёлые операции выполняются вне основного event loop Telegram.

## Ограничения

- OCR текста на кадрах пока отсутствует.
- Reel без распознаваемой речи нельзя озвучить или превратить в речевые субтитры.
- Входящие пользовательские MP4/MOV/MP3/WAV/M4A пока не обрабатываются.
- Официальный Telegram Bot API позволяет боту скачать входящий файл размером не более 20 МБ.
- Для больших входящих файлов требуется локальный Telegram Bot API или веб-загрузка.
- Состояние пользователей хранится в памяти и теряется после перезапуска процесса.

## Незавершённая настройка локального Telegram Bot API

VPS готов: Docker установлен, ресурсов достаточно.

Для запуска требуются параметры приложения с `my.telegram.org/apps`:

```env
TELEGRAM_API_ID=...
TELEGRAM_API_HASH=...
```

Сейчас `my.telegram.org` возвращает общую ошибку `ERROR`. Следующая попытка:

1. Подождать не менее 24 часов после частых попыток.
2. Отключить VPN, iCloud Private Relay и блокировщики.
3. Использовать Chrome Incognito через мобильную точку доступа.
4. Проверить ограничения Telegram-аккаунта через `@SpamBot`.
5. Создать приложение и добавить `api_id/api_hash` напрямую в серверный `.env`.
6. Не отправлять `api_hash`, токен бота или содержимое `.env` в чат и Git.

Если регистрация продолжит падать, запасной вариант — защищённая веб-загрузка напрямую на VPS или в Yandex Object Storage.

## Как улучшить перевод

Сейчас применяется гибридная схема:

```text
Whisper
→ Yandex Translate (полный перевод без потери строк)
→ словарь Coach AI
→ YandexGPT (точечная редактура специальных терминов)
```

### Следующие улучшения

1. Перенести словарь терминов из Python-кода в SQLite/PostgreSQL.
2. Разделить словари по направлениям: бег, силовая подготовка, биомеханика, физиология, реабилитация.
3. Добавить проверенные пары оригинал/перевод и приоритет пользовательских исправлений.
4. Добавить кнопки `✅ Верно` и `✏️ Исправить` для терминов.
5. Хранить автора исправления, дату и число подтверждений.
6. Не разрешать модели автоматически закреплять собственные ответы без проверки тренером.
7. Передавать YandexGPT контекст соседних строк, но требовать вернуть то же число строк и таймкодов.
8. Отделять перевод от редакторского комментария и медицинских предположений.
9. Создать тестовый набор из 50–100 фрагментов с эталонным переводом.
10. Измерять полноту, сохранение чисел, терминологию и отсутствие выдуманных утверждений после каждого изменения.

### Рекомендуемые режимы

- `🌍 Дословный перевод` — Yandex Translate, максимальная полнота и скорость.
- `🧠 Тренерский перевод` — дословный перевод, проверенный словарь и точечная редактура YandexGPT.
- `📚 Учебная версия` — тренерский перевод с отдельными пояснениями терминов и ссылками на таймкоды.
